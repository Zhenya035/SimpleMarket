1. Связь "один-ко-многим"
    Внешний ключ всегда находится в зависимой сущности (в таблице "много").
    Пример: User (один) → Address (много). Внешний ключ UserId должен быть в сущности Address.
2. Связь "один-к-одному"
    Внешний ключ размещается в зависимой сущности (та, которая "не может существовать без главной").
    Главную сущность определите по смыслу. Например, User (главная) → Cart (зависимая).
    В главной сущности (User) не должно быть свойства CartId.
    В зависимой сущности (Cart) должно быть свойство UserId.
3. Связь "многие-ко-многим"
    Внешние ключи не указываются вручную.
    EF Core автоматически создаст промежуточную таблицу.
